# General

| Term | Definition |
| --- | --- |
| Feature spec | Written list that sets every rule and screen in a program. |
| Refresh (for feature spec) | Rewriting the list after plans change |
| Game engine | The code that runs game rules, points, and timers. |
| Local data layer | Code that saves and loads information on the same computer |
| Asset pipeline | The chain of steps that takes raw files and moves them into the program in the right size and format. |
| Regressions | Bugs that appear after a change, breaking something that used to work. |
| Type hints | Comments in Python code stating the variable type. |
| Full stack | Front-end and back-end development combined together |

# React

| Acronym | Term | Definition |
| --- | --- | --- |
|     | React | A JS library for building UIs |
| UI  | User interface | The point where users and computers interact; consists of separate components that are combined into a parent component to make a page. |
|     | Single page applications | Working with one single page template and simply updating all of the components; misleading as instead of being one single page, it is one single template |
| DOM | Document object model | The structure of HTML elements on a webpage |
|     | Component | Function (or class) that returns HTML; AKA JSX |
| JSX | JavaScript XML | Looks like HTML with slight syntax differences |
|     | (URL) Routing | Showing different pages based on the URL |
|     | Prop(erty) | Like function arguments, passes data down to child components |
|     | Prop drilling | Sending props down multiple layers, which can get messy |
